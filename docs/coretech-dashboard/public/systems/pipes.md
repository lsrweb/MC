# 管道与物流系统

## 结论：管道必须单独建模

管道不是普通方块。管道需要根据相邻连接动态改变形态，至少需要以下模型部件：

- 中心节点。
- 上下左右前后六方向连接段。
- 端口模型。
- 输入口/输出口颜色或纹理区分。
- 过滤器/阀门/红石控制模块。

## 管道类型

| 类型 | 功能 | 早期实现 | 后续升级 |
|---|---|---|---|
| 物品管道 | 搬运物品 | 相邻抽取/插入 | 过滤、优先级、速率升级 |
| 能量线缆 | 传输 FE | 点对点相邻传输 | 网络传输、损耗、电压等级 |
| 流体管道 | 搬运流体 | 暂缓 | 泵、阀门、储罐 |
| 物流节点 | 网络库存 | 暂缓 | 自动请求、自动合成 |

## 方块状态

管道需要记录六方向连接状态，例如：

- `north=true`
- `south=false`
- `up=true`
- `mode=extract/insert/passive`

Forge 里可以先用 blockstate 多部分模型，复杂后再考虑自定义 baked model。
